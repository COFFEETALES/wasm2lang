## v2026.04.111

### Added

- Constant-condition folding pass (`const-condition-folding`) in the `wasm2lang:codegen` optimize phase: folds `i32.eqz`/`i64.eqz` on a constant, always-false and always-true `br_if`, and side-effect-free `select` with constant discriminant. Per-function metrics exposed via `funcMetadata.constConditionFolds` keyed by fold category (`eqzConst`, `brIfNever`, `brIfAlways`, `selectFold`).
- `WASM2LANG_PROFILE` environment variable: when set, `PassRunner.runOnModule` accumulates per-pass wall-clock timings and flushes a `[wasm2lang profile]` report to stderr at module end. Zero overhead when off (lazy `process.env` read, single boolean check).
- Helper auto-discovery across all four backends: helper rosters are no longer duplicated in per-backend `identifiers.js`. A shared `emitOrCollectHelper_` funnel on `AbstractCodegen` reuses the emit path in a collect mode to derive the full helper name set, which feeds the mangler's reserved-names list automatically.
- `baseline` test-matrix variant in `scripts/wasm2lang_build_tests.sh` / `wasm2lang_run_tests.sh`: exercises every test with `binaryen:none` and without `wasm2lang:codegen`, giving a reference run to compare codegen-enabled output against. Per-test `.build.normalize` overrides for `wasm2lang_11_i64_ops` and `wasm2lang_17_codegen_passes` removed — the new variant subsumes them.
- Regression coverage in `wasm2lang_17_codegen_passes` for the unnamed value-typed root-block fix, the constant-condition fold, the do-while interior-back-branch rejection, and a new `eqz-or` compound-negation postbuild fixture.

### Changed

- JavaScript backend `BigInt`, `Number`, `BigInt.asIntN`, and `BigInt.asUintN` call sites now routed through mangler-aware alias helpers (`$w2l_bigint`, `$w2l_number`, `$w2l_bigint_asintn`, `$w2l_bigint_asuintn`), emitted as `var X = BigInt;` declarations so every reference survives identifier mangling with zero call-site overhead.
- JavaScript backend module shell function drops its inner name: `var module = function (stdlib, foreign, buffer) { ... }` instead of `var module = function javascriptModule (...) { ... }`.
- JavaScript backend `memory.fill`, `memory.copy`, and `memory.grow` rebuilt on native typed-array and resizable-`ArrayBuffer` APIs. `memory.fill` delegates to `TypedArray.prototype.fill`, `memory.copy` uses `Uint8Array.prototype.copyWithin` on a shared byte view, and `memory.grow` uses `ArrayBuffer.prototype.transfer` against a resizable buffer. Generated code no longer falls back to byte-loop helpers for bulk memory ops.
- `wasm2lang:codegen` shared state refactored into `src/backend/abstract_codegen/pass_state.js`: hosts `markHelper_` / `markBinding_` tracking, category constants, and the `emitOrCollectHelper_` / `emitHelpers_` funnel. Per-backend `identifiers.js` files reduced to thin mangler-profile wrappers.
- JavaScript backend helper-dependency graph now merges with the asm.js parent `HELPER_DEPS_` instead of replacing it, so inherited helpers keep their transitive dependencies.

### Fixed

- Unnamed value-typed block at function-body root: the tail expression is now returned as the function's implicit return instead of being stringified as a dangling expression statement (which fell through to the zero-value stabilizer). New shared helper `tryEmitRootValueBlock_` in `abstract_codegen/control_flow.js` handles the shape across all three backends.
- `labelN_` in `abstract_codegen/identifiers.js` now sanitizes raw binaryen label names (`.`, `$`, hyphens, etc.) when the identifier mangler is disabled. Previously emitted invalid identifiers in PHP and Java output when `--mangler` was unset.
- Loop-simplification do-while classification rejects loops whose body has an interior back-branch (unconditional `br $loop` or nested `br_if $loop` at shallower depth) before the trailing conditional. Such shapes would compile to `continue` in a `do { ... } while (cond)`, but a do-while's `continue` jumps to the cond check — an interior re-iterate would incorrectly exit when cond was false. The pass now falls through to for-loop classification or raw emission in that case.
