## v2026.06.113

### Added

- Anchor-backed metadata round-tripping for `wasm2lang:codegen` (`AnchorMarkers` + `w2l_codegen_meta` format v3): opaque `call $w2l_anchor` markers keep block metadata identifiable across binaryen optimization and binary round-trip, restoring `w2l_`-prefixed names and `br`/`br_table`/`switch` targets (v2 DFS-position scheme kept as a fallback).
- Single-pass/two-pass convergence: a combined normalize-and-emit run takes an internal `emitBinary`→`readBinary` round-trip (shared `recoverFromBinary_`) so its output is byte-identical to the split two-pass path.
- `TraversalKernel.forEachExpression`, a lightweight analysis-only walk, establishing analysis as a third traversal use-case.
- Mangler usage-discovery (`runUsageDiscovery_`): registers only the helpers and bindings the output actually references, freeing slots so hot identifiers reach the single-character mangling tier.
- Direct-labeled named blocks (`label: if (...)` instead of `label: { if (...) }`) and `stripForAssignment` removal of redundant RHS parens on assignments/stores.
- binaryen:max test variants `codegen_max`/`prenorm_max` (strict equivalence for binaryen:none, advisory for binaryen:max).

### Changed

- asm.js intish coercion-noise reduction: heap loads, pointer-offset sums, and chained `+`/`-` stay `intish` (`a+b+c|0`), with `|0` re-applied only where V8 requires it (`intish` conditions, `Math.imul`); top-level shifts shed redundant `|0`/`>>>0`.
- PHP i32 truncation deferred to consumers (bare `+`/`-`, folded masks, `I32_CLEAN_HELPERS_` skip, `!operand` for `eqz`); additive operands left unparenthesized under `+`; JS dispatch booleans via unary `+` and i64 Math intrinsics via mangler bindings.
- `br_table` dispatches with non-default case targets outside the nesting chain now flatten to a single switch (`break <ext>`), guarded against unrepresentable chains.
- Block-loop fusion (`label: for(;;)`) applied unconditionally; the phase-3 `optimize()` step is dropped, as its restructuring of multi-action `br_table` loops produced IR the codegen mishandled (QUIC handshake early-exit).
- Mangler registration reordered into hottest-first tiers (`getHotModuleBindings_`/`getAlwaysRegisteredBindings_`); Java infinite-loop trailing `break;` decided by the body's static wasm type.
- Bumped `binaryen` `^129`→`^130`, migrating `collectStaticMemory_` and the externs off the dropped `getNumMemorySegments`/`getMemorySegmentInfo` onto the data-segment APIs; output unchanged.
- Removed the dead `ConstConditionFoldingPass`/`DropConstElisionPass` and other unused helpers, plus broad internal deduplication with no output change (positional i64/LEB arrays for Closure mangling, centralized renderers/walkers/op-tables).

### Fixed

- Kernel leave callbacks now see post-`REPLACE_NODE` child slots (`walkExpression` re-fetches `nodeCtx.expression` after walking children), fixing silently-dropped nested transformations; covered by the `kernel-leave-freshness` family.
- asm.js validation: don't direct-label an `if` with an `else` arm (V8 rejects the `else`-arm break), coerce the RHS of helper-routed sub-aligned i32 stores, and resolve switch-epilogue chain-breaks to a real label instead of `break $_;`.
- Local-init folding overrides re-derived after the binary round-trip (`reanalyzeOverrides`), since binaryen renumbers locals on write.
- Trap-helper liveness no longer false-matches a single-character mangled name, and JS memory helpers no longer fall back to verbatim `$w2l_memory_*` identifiers.
- Removed a duplicate `metadata_section.js` entry from the compile manifest.
