## v2026.04.112

### Fixed

- JavaScript backend `emitMetadata` crash on modules that declare no memory. The resizable-`ArrayBuffer` cap path now guards `wasmModule.getMemoryInfo()` with `hasMemory()` and falls back to `heapSize * 16` when no memory is declared, mirroring the guarded pattern already used by `resolveHeapSize_`.

### Changed

- README rewritten for clearer positioning and faster onboarding: concise Quick start up top, condensed motivation bullets, unified CLI essentials table, and explanatory heap-size guidance that documents the per-backend `--define` keys (`ASMJS_HEAP_SIZE`, `JS_HEAP_SIZE`, `PHP64_HEAP_SIZE`, `JAVA_HEAP_SIZE`) and the default derived from the module's declared `memory.initial * 65536`. Output described as "native source code" rather than "human-readable" to reflect that emitted code is mangled and coercion-typed for direct host-toolchain compilation, not for reading.
